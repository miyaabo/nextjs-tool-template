# Project Principles

## 目的
このプロジェクトは、小型Webツールを短期間で量産し、低コストで公開しながら、収益化の可能性を検証することを目的とする。

## 初月の前提
- 初月目標は月100円
- 固定費は極力0円に近づける
- 赤字を出さないことを重視する
- 収益最大化ではなく、当たる型の発見を優先する

## 絶対原則
- 公開を優先する
- 最小構成から考える
- 1本の完成度より、量産しやすい型を重視する
- 小型ツールを短期間で出す
- 同じテンプレで横展開できる構成を目指す

## 技術原則
- フロントエンドのみで成立する案を優先する
- DBは原則使わない
- ログイン機能は原則入れない
- バックエンドは原則入れない
- 外部API必須の構成は避ける
- 静的ホスティングで公開できる形を優先する

## ツール原則
- 1画面で完結しやすいものを優先する
- 入力してすぐ結果が出るものを優先する
- 実用性があることを重視する
- 数時間〜1日程度で試作できるものを優先する
- 後から派生ツールを作りやすい構造を重視する

## 収益化原則
- 初月は広告収益を主目的にしない
- まず無料で使われる価値があるかを見る
- 反応があったものにだけ有料導線を後付けする
- 小額課金、note導線、アフィリエイト導線などを検討対象とする

## 判断に迷ったとき
迷った場合は、常に以下を優先すること。
1. 固定費が低い
2. 早く公開できる
3. 実装が軽い
4. 量産に流用できる
5. 実際に使われそう
