# Prompt Templates

この文書は、AIに小型Webツールの企画・設計・実装を依頼するときのテンプレート集である。
毎回この全文を渡すのではなく、必要なテンプレートだけ使うこと。

## 1. 新規ツール案を出させるテンプレ

以下の条件で、新しい小型Webツール案を10個提案してください。

条件:
- 初月向き
- 固定費ほぼ0円で公開可能
- フロントエンドのみで成立しやすい
- 1画面で完結しやすい
- 数時間〜1日以内で試作可能
- 実用性がある
- 同じ型で横展開しやすい
- 後から収益導線を足しやすい

出力形式:
1. ツール名
2. 一言説明
3. 想定ユーザー
4. 入力内容
5. 出力内容
6. 初月向きな理由
7. 派生ツール案

制約:
- バックエンド前提の案は除外
- ログイン必須の案は除外
- DB前提の案は除外
- AI API常用前提の案は除外

## 2. ツール案を具体化させるテンプレ

以下のツール案を、実装できるレベルまで具体化してください。

対象ツール:
[ここにツール名]

整理してほしい内容:
1. ツールの目的
2. 想定ユーザー
3. ユーザーが入力する項目
4. ツールが返す結果
5. 1画面でのUI構成
6. 最小実装で必要な機能
7. 今回は入れない機能
8. 将来追加できる機能
9. 収益導線候補
10. 実装上の注意点

出力ルール:
- 最小構成から考える
- 初月に不要な要素は明確に除外する
- 1画面完結を前提にする
- 実装しやすい順で整理する

## 3. MVPだけに絞らせるテンプレ

以下のツールについて、初月向けのMVPだけを定義してください。

対象ツール:
[ここにツール名]

やってほしいこと:
- 最小機能だけに絞る
- 必須機能と不要機能を分ける
- 最短で公開できる構成にする
- UIは1画面前提にする

出力形式:
1. MVPの目的
2. 必須機能
3. 不要機能
4. 画面構成
5. 必要な入力欄
6. 必要なボタン
7. 結果表示内容
8. 公開可能になる条件

制約:
- ログインなし
- DBなし
- バックエンドなし
- APIなしでも成立する形を優先

## 4. 共通テンプレートを作らせるテンプレ

小型Webツールを量産するための共通UIテンプレートを設計してください。

目的:
- 複数ツールに使い回せること
- 実装が軽いこと
- スマホでも使えること
- 1画面で完結すること

必要な要素:
- ヘッダー、ツール名、ツール説明
- 入力フォームエリア
- 実行ボタン、リセットボタン
- 結果表示エリア
- コピーボタン
- 使用例、注意事項
- 関連ツールリンク欄

## 5. 既存テンプレから派生案を出させるテンプレ

以下の既存テンプレートをベースに、新しい派生ツール案を5個作ってください。

ベーステンプレ:
[ここにテンプレ名や既存ツール名]

条件:
- UI構成は大きく変えない
- 入力項目と出力内容の差し替えで成立しやすい
- 同じ実装資産を流用しやすい
- 初月向きの軽量構成を維持する
